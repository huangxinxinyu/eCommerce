# 高性能秒杀系统

## 项目概述

基于微服务架构的高性能秒杀电商平台。系统通过多级缓存、Redis原子操作和异步消息队列处理限时秒杀活动中的海量并发请求。

## 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   API网关   │────▶│   用户服务   │     │   支付服务   │
└─────────────┘     └─────────────┘     └─────────────┘
                          │                    │
                          ▼                    │
                    ┌─────────────┐            │
                    │   库存服务   │◀───────────┘
                    │  (核心模块)  │
                    └─────────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
   ┌──────────┐    ┌────────────┐    ┌──────────┐
   │   Redis  │   │  RocketMQ  │    │  MySQL   │
   └──────────┘   └────────────┘    └──────────┘
```

## 核心解决方案

### 1. 多级缓存架构

```
请求 → 布隆过滤器 → Caffeine(L1) → Redis(L2) → MySQL(L3)
```

- **布隆过滤器**: 过滤无效商品ID，防止缓存穿透
- **Caffeine**: 进程内热点数据缓存（上限1万条，TTL 30秒）
- **Redis**: 分布式缓存，存储库存和用户购买记录

### 2. 原子性库存操作（Redis Lua脚本）

seckill.lua: 原子扣减库存 + 限购校验

```lua
-- 1. 检查用户是否已购买
-- 2. 检查库存是否充足
-- 3. 原子操作：扣减库存 + 标记用户已购买
-- 返回：1=成功，0=库存不足，-1=已购买
```

- Lua脚本保证库存操作的原子性，杜绝超卖
- 确保一人一单的限购策略

### 3. 异步订单处理

```
用户请求 → 消息队列 → 订单服务 → 创建订单 → 等待支付
```

- 立即返回订单号，提升用户体验
- 通过RocketMQ流处理实现订单异步创建
- 解耦秒杀峰值与订单处理流程

### 4. 类TCC库存管理

- **Try阶段**: 在Redis中预占库存（Lua原子操作）
- **Confirm阶段**: 支付成功后实际扣除MySQL库存
- **Cancel阶段**: 取消/超时后通过MQ恢复Redis库存

### 5. 分布式锁（Redisson）

- 订单支付更新：防止重复支付
- 订单超时关闭：确保只有一个执行者
- 库存回滚确认：防止重复回滚

## 核心模块

| 模块 | 职责 |
|------|------|
| `ecommerce-gateway` | 限流、请求路由 |
| `ecommerce-user` | 用户注册、认证 |
| `ecommerce-store` | 商品目录、秒杀核心逻辑、库存管理 |
| `ecommerce-order` | 订单生命周期、超时关闭、取消 |
| `ecommerce-pay` | 支付创建、回调处理 |
| `ecommerce-email` | 异步通知（模拟） |

## 消息流转

| Topic | 用途 |
|-------|------|
| `seckill-order` | 异步创建订单 |
| `order-delay` | 15分钟超时关闭调度 |
| `pay-success` | 支付成功通知订单服务 |
| `stock-rollback` | 取消/超时时的库存回滚 |
| `pay-confirmed` | 支付确认后实际扣减库存 |
