## 3. 服务职责（本地版）

### 3.1 Gateway Service

**当前实现**：
- 路由转发到后端服务
- CORS 跨域处理
- 简单的请求参数校验

**暂不实现**：
- 滑动窗口限流（后续加）

### 3.2 Store Service

**核心功能**：
- 秒杀商品查询
- **Lua 脚本库存预扣**（原子操作）
- **一人一单校验**（Redis Set）

**库存预扣流程**：
```
1. 检查用户是否已购买 → Redis SISMEMBER
2. Lua 脚本原子扣减库存 + 标记用户 → Redis EVAL
3. 发送订单创建消息 → RocketMQ
4. 返回下单成功
```

### 3.3 Order Service

**核心功能**：
- 消费 MQ 消息创建订单
- 订单状态管理
- **延迟消息关单**（15分钟）

**关单流程**：
```
1. 订单创建时发送延迟消息
2. 延迟消息到期后查询订单状态
3. 未支付 → 回滚库存 + 更新状态
4. 已支付 → 跳过
```

### 3.4 Pay Service

**核心功能**：
- 接收支付请求
- 模拟银行处理（延迟 1-3 秒）
- 回调通知 Order Service

**注意**：
- 不做幂等控制（后续加）
- 不做分布式锁（后续加）

### 3.5 Email Service

**核心功能**：
- Log 打印模拟通知
- 不做真实短信/邮件

## 4. Redis 数据结构

### 4.1 库存预扣

```
Key: stock:seckill:{skuId}
Type: String (数字)
Value: 可预扣库存数量
```

### 4.2 一人一单

```
Key: user:bought:{skuId}
Type: Set
Member: userId
```

### 4.3 Lua 脚本（核心）

```lua
-- seckill.lua
-- KEYS[1]: 库存 Key
-- KEYS[2]: 用户购买记录 Key
-- ARGV[1]: 购买数量
-- ARGV[2]: userId

-- 1. 检查用户是否已购买
local alreadyBought = redis.call('SISMEMBER', KEYS[2], ARGV[2])
if alreadyBought == 1 then
    return -1  -- 已购买
end

-- 2. 检查并扣减库存
local stock = tonumber(redis.call('GET', KEYS[1]))
if stock < tonumber(ARGV[1]) then
    return 0  -- 库存不足
end

-- 3. 原子扣减库存
redis.call('DECRBY', KEYS[1], ARGV[1])

-- 4. 标记用户已购买
redis.call('SADD', KEYS[2], ARGV[2])

return 1  -- 成功
```

## 5. RocketMQ Topic 设计

| Topic | Tag | 用途 | 消费者 |
|-------|-----|------|--------|
| seckill-order | create | 订单创建 | Order Service |
| order-close | delay | 延迟关单 | Order Service |
| order-pay | callback | 支付回调 | Order Service |

### 5.1 消息流程

```
用户下单
    ↓
Store Service 发送消息 → seckill-order
    ↓
Order Service 消费 → 创建订单
    ↓
Order Service 发送延迟消息 → order-close (15分钟后)
    ↓
用户支付
    ↓
Pay Service 回调 → order-pay
    ↓
Order Service 更新状态为已支付
    ↓
用户未支付
    ↓
order-close 消息到期 → 检查状态 → 未支付则关单+回滚库存
```

## 7. 核心流程（本地开发实现顺序）

### 7.1 Phase 1：基础数据层

**目标**：搭建基础环境，数据能读写

**任务**：
1. Docker Compose 启动 MySQL、Redis、Nacos、RocketMQ
2. 创建数据库和表结构
3. 初始化秒杀商品数据
4. 实现商品查询接口

### 7.2 Phase 2：秒杀核心

**目标**：实现库存预扣和一人一单

**任务**：
1. 实现 Lua 脚本库存预扣
2. 实现一人一单校验
3. 实现秒杀下单接口
4. 实现库存回滚接口

### 7.3 Phase 3：订单与MQ

**目标**：异步解耦，实现订单自动关单

**任务**：
1. 配置 RocketMQ
2. 实现订单创建消费者
3. 实现延迟消息关单
4. 实现订单查询接口

### 7.4 Phase 4：支付流程

**目标**：完成支付和回调流程

**任务**：
1. 实现支付接口
2. 实现支付回调

**任务**：

3. 添加熔断降级
